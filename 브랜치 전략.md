
#### 해당 토이프로젝트는 본인의 1인용 장난감 프로젝트지만, 간략하게나마 git-flow 브랜치 전략을 사용함.

```
feature/이슈번호 -> 해당 이슈에 맞게 작업
feature/해당 프로그램명 -> 해당 프로그램에 대한 작업 결과 검토 후 반영
develop -> 결과 테스트용 브랜치
release -> 실제 배포 전 브랜치
master -> 라이브 서버에 제품으로 출시될 브랜치
```
